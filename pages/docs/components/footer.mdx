# Componente: Footer

O `Footer` é um componente global exibido na parte inferior de todas as páginas do **DEX BLOG**. Ele serve para apresentar informações essenciais de direitos autorais, a versão atual do software e links importantes.

## Funcionalidades Principais

1.  **Ano Dinâmico:** O ano exibido nos direitos autorais é atualizado automaticamente, garantindo que a informação esteja sempre correta sem a necessidade de intervenção manual.
2.  **Exibição de Versão Sincronizada:** A versão do software (`vX.X.X`) é lida diretamente do campo `"version"` no arquivo `package.json` do projeto. Isso garante que a versão exibida no site esteja sempre sincronizada com a versão oficial do código.
3.  **Estrutura Simples e Reutilizável:** Por ser um componente isolado, pode ser facilmente importado e integrado a qualquer layout de página principal.

---

## Código-Fonte do Componente

Este é o código-fonte principal do componente, localizado em `src/components/Footer.tsx`.

```tsx
// src/components/Footer.tsx

import packageJson from '../../package.json';

export function Footer() {
  const currentYear = new Date().getFullYear();
  const version = packageJson.version;

  return (
    <footer className="z-50 w-full border-t border-gray-200 bg-white shadow-sm dark:border-gray-800 dark:bg-gray-950">
      <div className="container mx-auto flex h-16 w-full max-w-6xl items-center justify-center px-4 md:px-6">
        <div className="text-center text-sm text-gray-500 dark:text-gray-400">
          © {currentYear} DEX BLOG | v{version}
        </div>
      </div>
    </footer>
  );
}
```

---

## Dependências

-   **`package.json`:** O componente depende diretamente deste arquivo na raiz do projeto para obter o número da versão. Qualquer atualização no campo `version` será refletida automaticamente no `Footer`.

---

## Como Usar

Para utilizar o componente `Footer`, importe-o e adicione-o ao layout principal da sua aplicação, geralmente no arquivo `src/app/layout.tsx`.

```tsx
// Exemplo de uso em src/app/layout.tsx

import { Footer } from '@/components/Footer';

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <div className="flex min-h-screen flex-col">
          <main className="flex-grow">
            {children}
          </main>
          <Footer />
        </div>
      </body>
    </html>
  );
}
```

## Boas Práticas e Manutenção

- **Versionamento:** A versão deve ser atualizada seguindo as práticas de Versionamento Semântico (SemVer). Ao lançar uma nova versão, atualize o `package.json` e crie a tag Git correspondente (ex: `git tag v1.5.0`).
- **Não Modificar Diretamente:** Evite "hardcodar" valores de ano ou versão diretamente no componente. A lógica atual foi projetada para ser automática e livre de manutenção.
